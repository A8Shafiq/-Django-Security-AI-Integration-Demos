🌐 DOM-based XSS (Client-Side)

Malicious scripts injected entirely via JavaScript — no server involvement

🔍 How DOM-based XSS Works

STEP 1: Attacker crafts a URL with a malicious hash fragment (e.g., #<img src=x onerror=alert('XSS')>)

STEP 2: Victim clicks the link — the hash fragment is NOT sent to the server

STEP 3: JavaScript on the page reads location.hash

STEP 4: VULNERABLE: innerHTML parses the value as HTML → script executes

STEP 4: SECURE: textContent treats it as plain text → no execution

Key difference: The malicious payload never reaches the server. It's processed entirely by client-side JavaScript.

🔴 Vulnerable (innerHTML)

Uses innerHTML — browser parses and executes HTML/JS

Output:

// ❌ DANGEROUS CODE
var input = document.getElementById('vuln-input').value;
element.innerHTML = input;
// Browser PARSES HTML → script executes!

🟢 Secure (textContent)

Uses textContent — browser treats everything as plain text

Output:

// ✅ SAFE CODE
var input = document.getElementById('safe-input').value;
element.textContent = input;
// Browser renders as PLAIN TEXT only

DOM-based XSS often uses location.hash. Try adding a hash to the URL:

The page will automatically read the hash and inject it into both panels when loaded.

🛡️ Mitigation Techniques for DOM-based XSS

  • Use textContent instead of innerHTML: textContent never parses HTML — it's always safe.
  • Avoid dangerous DOM APIs: innerHTML, outerHTML, document.write(), eval()
  • Sanitize before DOM insertion: Use DOMPurify library: element.innerHTML = DOMPurify.sanitize(userInput)
  • Content Security Policy: Content-Security-Policy: script-src 'self' blocks inline scripts
  • Input validation in JS: Validate/sanitize user input before using it in DOM operations
// ❌ DANGEROUS — All of these can execute scripts:
element.innerHTML = userInput;
element.outerHTML = userInput;
document.write(userInput);
eval(userInput);

// ✅ SAFE — These treat input as plain text:
element.textContent = userInput;
element.innerText = userInput;
element.setAttribute('value', userInput);

// ✅ SANITIZE — Use DOMPurify if you MUST render HTML:
// <script src="https://cdnjs.cloudflare.com/ajax/libs/dompurify/3.0.6/purify.min.js"></script>
element.innerHTML = DOMPurify.sanitize(userInput);
