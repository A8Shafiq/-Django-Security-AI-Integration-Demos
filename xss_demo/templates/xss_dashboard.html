⚡ Cross-Site Scripting (XSS) Demo

Learn how XSS attacks work and how to prevent them — 3 types demonstrated

💾

Stored XSS

Malicious script is permanently stored in the database (e.g., in a comment). Every visitor who views the page gets attacked. The most dangerous type.

HIGH SEVERITY
🔄

Reflected XSS

Malicious script is reflected from a URL parameter (e.g., a search query). Victim must click a crafted link. Most common XSS type.

MEDIUM SEVERITY
🌐

DOM-based XSS

Malicious script is injected entirely client-side via JavaScript (e.g., reading from URL hash). No server involvement.

CLIENT-SIDE

Feature Stored XSS Reflected XSS DOM-based XSS
