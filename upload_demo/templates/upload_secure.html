🛡️ Secure File Upload

✓ PROPERLY VALIDATED

This upload endpoint applies multiple layers of validation before accepting a file. Each check mitigates a specific attack vector. Upload a file to see every validation step in action.

Security Checks Applied

  • Extension whitelist — only .jpg .jpeg .png .gif .pdf .txt allowed
  • MIME-type whitelist — Content-Type must match allowed media types
  • File size limit — maximum 2 MB per file
  • Filename renamed — UUID replaces original name (prevents path traversal)
1

Extension Whitelist

Only files with known safe extensions are accepted. This blocks executable scripts like .php, .exe, .sh. The check uses a strict set — no pattern matching that could be bypassed.

2

MIME-Type Validation

The HTTP Content-Type header is checked against a whitelist. While MIME types can be spoofed, this adds defense-in-depth. In production, you'd also check the file's magic bytes.

3

File Size Limit

Files larger than 2 MB are rejected. This prevents disk exhaustion DoS attacks and limits bandwidth abuse. In production, limits are also enforced at the web server (nginx) level.

4

UUID Rename

The original filename is discarded and replaced with a random UUID like a1b2c3d4...ext. This prevents path traversal (../../), file overwrites, and information leakage from predictable names.
